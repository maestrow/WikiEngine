## 26.07.2014

#### Соглашение о получении/кешировании данных с сервера между страничками list, item, edit

Есть три представления: list (просмотр списка), item (просмотр элемента), edit (редактирование элемента). При переходе от просмотра к редактированию и назад решено не использовать данные модели, т.к.:

- после просмотра страничку уже могли изменить
- в процессе редактирования могли изменить другую секцию статьи

Соответственно решено просто переходить по url, что вызывает получение данных с сервера.
